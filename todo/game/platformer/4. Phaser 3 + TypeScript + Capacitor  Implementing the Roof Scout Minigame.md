# Phaser 3 + TypeScript + Capacitor: Implementing the Roof Scout Minigame

## Overview

This document translates the council's recommended "Roof Scout" vertical climbing platformer minigame into concrete Phaser 3 + TypeScript implementation patterns. It covers every major subsystem: physics setup, touch controls, game-feel assists, level architecture, juice/feedback, guild stat integration, and Capacitor/Android performance. Code patterns are illustrative and intended to match the coding style already used across Clowder & Crest's existing minigame scenes.

***

## 1. Scene Configuration: Portrait Physics

The minigame should be a dedicated `Phaser.Scene` (e.g., `RoofScoutScene.ts`), matching the scene-per-minigame pattern already used in the game. The physics world should be tall and narrow, and the canvas must enforce portrait orientation.[^1]

```typescript
// capacitor.config.ts
// orientation: 'portrait' in Capacitor config + CSS height: 100vh, touch-action: none

// RoofScoutScene.ts
const GAME_WIDTH = 360;
const GAME_HEIGHT = 740; // 2:1 portrait ratio, matches Android viewport

export default class RoofScoutScene extends Phaser.Scene {
  constructor() { super({ key: 'RoofScoutScene' }); }

  create() {
    // Extend physics world taller than the screen to hold pre-generated platforms
    this.physics.world.setBounds(0, 0, GAME_WIDTH, GAME_HEIGHT * 4);

    // Arcade physics with realistic but readable gravity
    // ~1200 feels weighty but fair; tune to taste
    this.physics.world.gravity.y = 1200;
  }
}
```

Phaser's Scale Manager `FIT` mode keeps the canvas proportional across Android screen sizes:[^2]

```typescript
scale: {
  mode: Phaser.Scale.FIT,
  autoCenter: Phaser.Scale.CENTER_BOTH,
  width: GAME_WIDTH,
  height: GAME_HEIGHT,
}
```

For the Capacitor APK, lock orientation via the `@capacitor/screen-orientation` plugin so players are never asked to rotate:[^2]

```typescript
import { ScreenOrientation } from '@capacitor/screen-orientation';
ScreenOrientation.lock({ orientation: 'portrait' });
```

## 2. Player Physics Body

Arcade physics is the right engine here — not Matter.js. Arcade is simpler, more predictable, easier to tune for fairness, and avoids the ghost-collision tile edge problems that afflict Matter. The cat sprite should have a narrow rectangular body that matches its visual trunk, not the full bounding box.[^3]

```typescript
// In create()
this.player = this.physics.add.sprite(GAME_WIDTH / 2, startY, 'cat-atlas', 'idle-0');
this.player.setCollideWorldBounds(true);
this.player.body.setSize(22, 36);   // narrower than sprite for forgiveness
this.player.body.setOffset(5, 8);

// Prevent platform edge snagging — set max velocity
this.player.body.setMaxVelocity(220, 900);

// Light drag so horizontal momentum bleeds off after a jump
this.player.body.setDrag(400, 0);
```

`body.blocked.down` is the canonical Arcade check for whether the cat is grounded. Use it everywhere — it handles tile collisions properly and resets each frame.[^4][^5]

## 3. Two-Zone Touch Controls

Avoid virtual d-pads and joysticks entirely. The recommended control scheme is a **split-screen two-zone tap** (left half = jump/push left, right half = jump/push right) with a hold-to-cling/slide option. This uses the bottom half of the portrait screen — the ergonomically natural thumb zone.[^6][^2]

```typescript
// In create()
// Enable multi-touch — Phaser defaults to 1 pointer
this.input.addPointer(2); //

// Create two invisible interactive zones
const leftZone = this.add.zone(0, 0, GAME_WIDTH / 2, GAME_HEIGHT)
  .setOrigin(0, 0)
  .setInteractive()
  .setScrollFactor(0); // fixed to camera

const rightZone = this.add.zone(GAME_WIDTH / 2, 0, GAME_WIDTH / 2, GAME_HEIGHT)
  .setOrigin(0, 0)
  .setInteractive()
  .setScrollFactor(0);

// Track which zone is held
let leftHeld = false;
let rightHeld = false;

leftZone.on('pointerdown', () => { leftHeld = true; this.attemptJump('left'); });
leftZone.on('pointerup', () => { leftHeld = false; });
rightZone.on('pointerdown', () => { rightHeld = true; this.attemptJump('right'); });
rightZone.on('pointerup', () => { rightHeld = false; });

// Expose to update() loop
this.leftHeld = leftHeld;
this.rightHeld = rightHeld;
```

Using `pointer.x < GAME_WIDTH / 2` in `pointerdown` is an alternative that's equally reliable and avoids two Zone objects.[^7][^8]

### Variable Jump Height (Hold to Jump Higher)

Implement variable jump height by cutting the upward velocity when the player releases the tap early, matching how Super Mario Run handles it:[^9][^10]

```typescript
// In update()
// If jumping upward and the tap was released — cut velocity
if (this.player.body.velocity.y < 0 && !this.leftHeld && !this.rightHeld) {
  // Reduce upward speed to 40% of current — creates "tap = small jump, hold = big jump"
  const vel = this.player.body.velocity.y;
  if (vel < -200) { // only cut if still meaningfully airborne
    this.player.setVelocityY(vel * 0.4);
  }
}
```

## 4. Coyote Time and Jump Buffering

These two "hidden assists" are industry-standard for fair, responsive platformer feel and are especially important on touch where imprecise timing is unavoidable.[^11]

**Coyote Time** lets the player still jump for a short window after walking off a ledge:

```typescript
private coyoteTimeMs = 120; // 120ms is a generous but not cheaty window
private lastGroundedTime = 0;

// In update()
if (this.player.body.blocked.down) {
  this.lastGroundedTime = this.time.now;
}

attemptJump(direction: 'left' | 'right') {
  const coyoteOk = (this.time.now - this.lastGroundedTime) < this.coyoteTimeMs;
  const canJump = this.player.body.blocked.down || coyoteOk || this.isWallClinging;

  if (canJump) {
    this.doJump(direction);
    this.lastGroundedTime = 0; // consume coyote window
  } else {
    // Record a jump buffer request
    this.jumpBufferTime = this.time.now;
    this.jumpBufferDirection = direction;
  }
}
```

**Jump Buffering** registers a jump press slightly before the cat lands and fires it the instant they touch down:

```typescript
private jumpBufferMs = 150; // 150ms lookahead
private jumpBufferTime = 0;
private jumpBufferDirection: 'left' | 'right' | null = null;

// In update(), after collision resolution
if (this.player.body.blocked.down && this.jumpBufferTime > 0) {
  if ((this.time.now - this.jumpBufferTime) < this.jumpBufferMs) {
    this.doJump(this.jumpBufferDirection!);
  }
  this.jumpBufferTime = 0;
  this.jumpBufferDirection = null;
}
```

## 5. Wall Cling and Wall Jump

Wall cling is the most "cat-like" mechanic and differentiates this from the existing courier runner. Arcade Physics `body.blocked.left` and `body.blocked.right` detect wall contact.[^5][^4]

```typescript
// In update()
const touchingLeft = this.player.body.blocked.left;
const touchingRight = this.player.body.blocked.right;
const onGround = this.player.body.blocked.down;

this.isWallClinging = (touchingLeft || touchingRight) && !onGround;

if (this.isWallClinging) {
  // Slow the fall — "cling" feel, cat gripping stone
  const clampedFall = Math.min(this.player.body.velocity.y, 80); // max 80px/s fall while clinging
  this.player.setVelocityY(clampedFall);
  // Play cling animation
  this.player.anims.play('cat-cling', true);
}
```

Wall jump launches the cat away from the wall in a diagonal arc:[^12][^13]

```typescript
doJump(direction: 'left' | 'right') {
  const jumpForce = -620;
  const wallKickX = 200; // horizontal push away from wall

  if (this.isWallClinging) {
    // Jump off wall
    const pushDir = this.player.body.blocked.right ? -1 : 1; // away from wall
    this.player.setVelocityY(jumpForce);
    this.player.setVelocityX(pushDir * wallKickX);
  } else {
    // Standard ground or coyote jump with directional lean
    this.player.setVelocityY(jumpForce);
    const leanX = direction === 'left' ? -100 : 100;
    this.player.setVelocityX(leanX);
  }
}
```

The directional lean from a standard jump (+/- 100 X velocity) is subtle but makes the cat feel agile rather than just bouncing straight up. For the wall-jump, using `setAccelerationX` instead of `setVelocityX` with a high acceleration and a capped max speed gives more satisfying momentum.[^13]

## 6. Camera: Upward-Scrolling Portrait

The camera must scroll upward tracking the cat, showing what's above (upcoming hazards) rather than behind. Keep the cat in the lower third of the screen so the player can see the next 2–3 platforms.[^14]

```typescript
// In create()
this.cameras.main.setBounds(0, 0, GAME_WIDTH, GAME_HEIGHT * 4);
this.cameras.main.startFollow(this.player, true);

// Offset: keep cat ~70% down the screen so upward path is visible
this.cameras.main.setFollowOffset(0, GAME_HEIGHT * 0.25); //

// Lerp for smooth camera that doesn't snap, doesn't lag
this.cameras.main.setLerp(1.0, 0.08); // instant X, smooth Y follow
```

For parallax depth — medieval stone walls scrolling slower than rooftop beams — use `tileSprite` with different scroll factors:[^15][^16][^17]

```typescript
// Background stone wall layer — moves at 30% camera speed (parallax depth)
this.bg = this.add.tileSprite(0, 0, GAME_WIDTH, GAME_HEIGHT, 'wall-bg')
  .setOrigin(0, 0)
  .setScrollFactor(0, 0.3); // only Y scroll at 0.3x camera rate
```

HUD elements (fish counter, height progress) must be fixed to the viewport using `setScrollFactor(0)`:[^18][^17][^19]

```typescript
this.fishText = this.add.text(12, 12, '🐟 0', { fontSize: '18px' })
  .setScrollFactor(0)
  .setDepth(100);
```

## 7. Level Architecture: Modular Chunks

Rather than building a single long hand-authored tilemap, use a **modular chunk system** — a small library of hand-designed platform segments that are procedurally stacked vertically. This is the standard pattern for endless/finite vertical games in Phaser and makes difficulty scaling trivial: easier chunks early, harder later.[^20][^21][^22]

Each chunk is ~180px tall (roughly 2 screen segments). Design 15–20 chunks across 3 difficulty tiers.

```typescript
interface PlatformChunk {
  id: string;
  difficulty: 1 | 2 | 3;
  platforms: { x: number; y: number; width: number; type: 'solid' | 'crumbling' | 'moving' }[];
  collectibles?: { x: number; y: number; type: 'fish' | 'scroll' }[];
  hazards?: { x: number; y: number; type: 'spike' | 'oil-slick' }[];
  wallGaps?: { side: 'left' | 'right'; y: number; height: number }[]; // openings for wall jumps
}

// Spawn chunks as the player ascends
private spawnNextChunk() {
  const tier = this.getChunkTier(); // 1→2→3 based on height progress
  const eligible = CHUNKS.filter(c => c.difficulty <= tier);
  const chunk = Phaser.Math.RND.pick(eligible);
  this.buildChunk(chunk, this.nextChunkY);
  this.nextChunkY -= 180; // move up for next chunk
}
```

Recycle chunks below the camera using object pooling to keep mobile memory flat:[^23][^2]

```typescript
// In update()
this.platformGroup.getChildren().forEach((p: Phaser.GameObjects.GameObject) => {
  const sprite = p as Phaser.Physics.Arcade.Sprite;
  if (sprite.y > this.cameras.main.scrollY + GAME_HEIGHT + 200) {
    sprite.setActive(false).setVisible(false);
    this.platformGroup.killAndHide(sprite);
  }
});
```

## 8. "Juice" and Game Feel

"Juice" — the sensory feedback layer — does double duty: it rewards the player emotionally and delivers information about physics state. Every major event needs a visual and audio response.

### Squash and Stretch on Land

On landing, quickly squash the cat (squish tall-to-wide) then release back to normal. Use a Phaser tween chained on `blocked.down` transition:[^24][^25]

```typescript
private playLandSquash() {
  // Kill any existing scale tween
  this.tweens.killTweensOf(this.player);
  this.tweens.add({
    targets: this.player,
    scaleX: 1.35,
    scaleY: 0.70,
    duration: 60,
    ease: 'Sine.Out',
    yoyo: true,     // auto-reverses back to 1.0
    onComplete: () => { this.player.setScale(1, 1); }
  });
}

// In update() — detect landing transition
const justLanded = this.player.body.blocked.down && this.prevVelocityY > 200;
if (justLanded) {
  this.playLandSquash();
  this.sound.play('land-soft', { volume: 0.5 });
  this.spawnDustParticles(this.player.x, this.player.bottom);
}
this.prevVelocityY = this.player.body.velocity.y;
```

### Stretch on Jump

Stretch vertically on jump launch:

```typescript
private playJumpStretch() {
  this.tweens.add({
    targets: this.player,
    scaleX: 0.80,
    scaleY: 1.30,
    duration: 80,
    ease: 'Sine.Out',
    yoyo: true,
  });
}
```

### Particle Dust

Phaser 3.60+ uses `ParticleEmitter` directly. For older Phaser 3 versions, use `this.add.particles`:

```typescript
private dustEmitter: Phaser.GameObjects.Particles.ParticleEmitter;

// In create()
this.dustEmitter = this.add.particles(0, 0, 'dust-particle', {
  speed: { min: 20, max: 60 },
  angle: { min: 200, max: 340 },
  scale: { start: 0.5, end: 0 },
  lifespan: 300,
  quantity: 5,
  emitting: false,
});

spawnDustParticles(x: number, y: number) {
  this.dustEmitter.setPosition(x, y);
  this.dustEmitter.explode(6);
}
```

### Audio on Capacitor/Android

Android audio latency in WebView can run 200–400ms if warm-up is not handled. Pre-decode all audio in Phaser's preload, and call `this.sound.play(key)` as early as possible in scene creation to trigger warm-up:[^26][^27]

```typescript
// In preload()
this.load.audio('jump', ['assets/audio/jump.mp3', 'assets/audio/jump.ogg']);
this.load.audio('land-soft', ['assets/audio/land.mp3', 'assets/audio/land.ogg']);
this.load.audio('cling', ['assets/audio/cling.mp3', 'assets/audio/cling.ogg']);
this.load.audio('collect-fish', ['assets/audio/fish.mp3', 'assets/audio/fish.ogg']);

// In create() — warm up audio context with a silent play
this.sound.play('jump', { volume: 0 });
```

OGG + MP3 fallback ensures coverage across all Android WebView versions.[^2]

## 9. Guild Stat Integration

Cat stats should visibly change how the run feels — not just the reward multiplier — keeping the minigame tied to the management layer.

| Stat | Mechanical Effect | Implementation |
|------|------------------|----------------|
| **Agility** | Jump height and wall-cling duration | Scale `jumpForce` and `clampedFall` by agility ratio |
| **Weight/Size** | Fall speed (lighter = slower fall, easier cling) | Adjust `gravity.y` per-body via `this.player.body.setGravityY()` |
| **Courage** | How fast the "danger zone" rises (if using rising floor) | Scale `dangerZoneSpeed` |
| **Perception** | Reveal alternate routes or hidden fish locations | Pre-flag collectibles as visible/hidden on chunk build |
| **Bond** | Rare "co-op" moment: a guild mate appears mid-run and boosts a jump | Scripted event trigger at height milestone |

```typescript
// In create(), after receiving catStats from scene data
const stats = this.scene.settings.data.catStats as CatStats;

// Agility scales jump force — base 620, range 520–720
this.jumpForce = -620 + (stats.agility - 5) * 20;

// Weight affects per-body gravity override (not world gravity)
const extraGravity = (stats.weight - 5) * 30; // heavier cats fall faster
this.player.body.setGravityY(extraGravity);

// Wall cling duration scales with agility
this.clampedFallSpeed = 80 - (stats.agility - 5) * 8; // agile cats fall slower on walls
```

Pass stats through Phaser scene `data` on launch, matching the pattern used by BrawlScene and HuntScene:

```typescript
this.scene.start('RoofScoutScene', {
  catId: selectedCat.id,
  catStats: selectedCat.stats,
  jobId: currentJob.id,
  targetHeight: currentJob.targetHeight,
});
```

## 10. Run Structure and Clean Exit

Clowder & Crest's minigames "resolve cleanly" — they produce a clear outcome and return to the guild layer. The Roof Scout run needs:

- **Fixed height objective**: Each job specifies a `targetHeight` in pixels. Reaching it triggers the success sequence.
- **Rest perches as soft checkpoints**: If the cat falls to a rest perch below the last position, the player continues from there. A full fall to the starting Y ends the run with a partial reward.
- **No life system**: Falls cost progress, not retries. Short runs always resolve.

```typescript
// In update()
// Check win condition
if (this.player.y <= this.targetY) {
  this.triggerSuccess();
  return;
}

// Check rest perch checkpoint
this.restPerches.forEach(perch => {
  if (this.player.body.blocked.down && Math.abs(this.player.y - perch.y) < 4) {
    this.lastCheckpointY = perch.y;
  }
});

// Check out-of-bounds fall (fell past last checkpoint by too far)
if (this.player.y > this.lastCheckpointY + GAME_HEIGHT * 0.6) {
  this.triggerFall();
}
```

On success or fall, emit an event back to the job resolution system and stop the scene:

```typescript
triggerSuccess() {
  this.events.emit('minigame-result', {
    outcome: 'success',
    fishCollected: this.fishCollected,
    heightReached: this.startY - this.player.y,
    cleanRun: this.fishCollected === this.totalFish, // bonus flag
  });
  this.scene.stop();
}
```

## 11. Capacitor + Android Performance

### Rendering

Prefer `Phaser.CANVAS` over `Phaser.WEBGL` on lower-end Android devices. WebGL is faster when it works, but Capacitor's WebView on budget Android phones can stutter on WebGL context creation. Use `Phaser.AUTO` (auto-detects) and test on real devices.[^28][^2]

```typescript
type: Phaser.AUTO, // falls back to Canvas if WebGL issues arise
```

Reduce physics bodies on decorative tiles by using a single physics-enabled tilemap layer for collision geometry and placing visual decoration sprites **without** physics bodies.[^2]

### Object Pooling for Platforms and Collectibles

Pre-allocate all platforms at scene creation as a `StaticGroup` or `Group` with `maxSize`, then reuse rather than create/destroy:[^23][^2]

```typescript
// Pre-allocate 40 platform slots
this.platformGroup = this.physics.add.staticGroup({ maxSize: 40 });
this.fishGroup = this.physics.add.group({ maxSize: 30 });
```

### Texture Atlas

All Roof Scout sprites (cat frames, platform tiles, dust particles, collectibles) should be packed into a single atlas. One atlas = one draw call vs. 20+ individual PNGs:[^2]

```typescript
this.load.atlas('roof-scout-atlas', 'assets/roof-scout.png', 'assets/roof-scout.json');
```

### Integer Positions

Prevent subpixel rendering artifacts on Android by rounding sprite positions per frame:[^2]

```typescript
// In update()
this.player.x = Math.round(this.player.x);
this.player.y = Math.round(this.player.y);
```

### Memory: Scene Cleanup

Ensure the scene destroys all objects properly to prevent memory leaks across multiple minigame replays:[^28]

```typescript
// Override shutdown()
shutdown() {
  this.dustEmitter?.destroy();
  this.platformGroup?.clear(true, true);
  this.fishGroup?.clear(true, true);
  this.input.removeAllListeners();
}
```

## 12. Summary of Key Implementation Decisions

| Area | Recommendation | Rationale |
|------|---------------|-----------|
| Physics engine | Arcade, not Matter.js | Predictable, no ghost collisions, easily tunable for fairness |
| Control scheme | Two-zone tap (L/R half of screen) | No virtual buttons, one-handed portrait play, proven mobile pattern |
| Jump feel | Variable height via velocity cut on release | Rewards hold vs tap, mirrors Super Mario Run model |
| Game-feel assists | 120ms coyote time + 150ms jump buffer | Industry-standard for fair touch platforming |
| Wall mechanic | Cling (slow fall) + wall jump (opposite diagonal) | Distinctly cat-like, differentiates from lane-runner |
| Camera | Y-lerp follow, cat at 70% down screen | Upward planning view, portrait native |
| Level structure | Modular stacked chunks, 3 difficulty tiers | Scope-controlled, procedurally varied without hand-crafting long levels |
| Juice | Squash/stretch tweens + particle dust + audio warm-up | Communicates physics state, makes mobile touch feel responsive |
| Guild stats | Agility/weight/courage modify physics params | Keeps minigame tied to management layer |
| Capacitor | Phaser.AUTO, object pooling, single atlas, integer positions | Smooth 60fps on mid-range Android |
